## Reporting a Vulnerability

If you discover a security vulnerability within GITY, please do not file a public issue. Instead, please send an e-mail to the repository maintainers or use GitHub's private vulnerability reporting feature if enabled. 

All security vulnerabilities will be promptly addressed.

Please include the following information in your report:
- Type of vulnerability (e.g. XSS, SQLi).
- The location of the vulnerability (e.g. file name and line number, or URL).
- Any special conditions required to reproduce the issue.
- Step-by-step instructions to reproduce the issue.
- Proof-of-concept or exploit code (if possible).
- Potential impact of the vulnerability.

We will endeavor to respond to all reports within 48 hours.
